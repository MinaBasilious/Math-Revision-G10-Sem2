Row-Reduction to RREF

  1. Augmented matrix [  A∣b  ][\;A\mid \mathbf b\;][A∣b].

  2. Use elementary row operations (swap rows, multiply a row by nonzero scalar, add a multiple of one row to another) to reach Reduced Row-Echelon Form:

  • Leading 1s (“pivots”) each have zeros above and below.

  • Any row of all zeros is at the bottom.

  1. Once in RREF, you can read off solutions directly:

  • A pivot in every column of AAA ⇒ unique solution.

  • A zero row with nonzero entry in b\mathbf bb ⇒ no solution (inconsistent).

  • Fewer pivots than variables ⇒ infinitely many solutions (free variables).

  1. Linear Programming (LP)

Real-World Motivation

Suppose you run a factory making widgets and gadgets:

  • Profit: $50 per widget, $30 per gadget

  • Labor: widget = 2 hrs, gadget = 1 hr; total labor ≤ 100 hrs

  • Materials: widget uses 3 units, gadget uses 2 units; total materials ≤ 120 units

You want to decide how many of each to make to maximize profit without exceeding resources. That’s a classic LP problem.

Core Components

  1. Decision variables (x=x=x= # widgets, y=y=y= # gadgets).

  2. Objective function

Z=50x+30y(maximize). Z = 50x + 30y \quad\bigl(\text{maximize}\bigr). Z=50x+30y(maximize).

  1. Constraints (all linear inequalities)

{2x+1y  ≤  100(labor)3x+2y  ≤  120(materials)x  ≥0,y  ≥0(no negatives) \begin{cases} 2x + 1y \;\le\;100 &\text{(labor)}\\ 3x + 2y \;\le\;120 &\text{(materials)}\\ x\;\ge0,\quad y\;\ge0 &\text{(no negatives)} \end{cases} ⎩⎨⎧​2x+1y≤1003x+2y≤120x≥0,y≥0​(labor)(materials)(no negatives)​

  1. Feasible region: the set of all (x,y)(x,y)(x,y) satisfying the constraints—always a convex polygon (or unbounded region).

  2. Corner-Point Principle: the maximum or minimum of a linear objective over a convex polygon occurs at one of its vertices.
